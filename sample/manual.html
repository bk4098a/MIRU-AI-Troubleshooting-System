시스템 개요
MIRU AI 트러블슈팅 시스템이 무엇을 하는지 한눈에 이해합니다.
처리 흐름
① 신청서 작성
→
② Groq AI 스크립트 생성
→
③ Higgsfield 이미지 생성
→
④ 결과 확인
→
⑤ 관리자 검토
💡
전체 소요 시간: 신청 후 AI 생성까지 평균 20~60초. Groq LLM이 에러 상황을 분석해 Kling/Seedance용 영상 스크립트를 생성하고, Higgsfield가 PCOS 기기 이미지 3장(에러 상황 · 해결 1단계 · 정상 복귀)을 자동 생성합니다.

Step 3 — 제품 & 에러 유형 선택
국가 선택 후 제품을 선택하고, 발생한 에러 유형 칩을 클릭합니다. 여러 개 선택 가능하며, 목록에 없는 경우 직접 입력할 수 있습니다.
화면 안내

    파란색 칩 = 선택됨. 흰색 = 미선택. 여러 개 동시 선택 가능.
💡
목록에 없는 에러: 칩 영역 하단의 "+ 직접 입력" 버튼을 눌러 텍스트를 직접 입력할 수 있습니다. 직접 입력한 값은 AI 스크립트 생성에 그대로 반영됩니다.
ACM의 경우: Scanner / Printer / Camera / USB / Battery 탭이 별도로 표시됩니다.
⚠
AI 프리셋 이미지: PCOS 제품의 11개 주요 에러(Paper Jam, UIK Mismatch, Flash Card Mismatch 등)는 사전 생성된 고품질 이미지를 즉시 표시합니다. 프리셋 외 에러는 실시간 AI 생성됩니다.

Step 4 — 에러 상세 설명 (3단계)
에러 상황을 3개 항목으로 순서대로 작성합니다. 이전 항목을 5자 이상 작성해야 다음 항목이 활성화됩니다.
화면 안내
submit.html
  • 1
    A 완료 — 녹색으로 완료 표시됩니다.
  • 2
    B 활성화 — A 작성 후 B 입력란이 나타납니다.
  • 3
    C 잠김 — B 완료 전까지 흐리게 표시됩니다.
항목입력 내용예시최소 길이
A. 에러 발생 상황필수현재 화면에 표시되는 에러 메시지, 기기 상태"PCOS 화면에 'Paper Jam' 메시지 표시. 투표 진행 불가."5자
B. 에러 발생 전 조작필수에러 발생 직전에 한 조작 내용"투표용지를 삽입하려다가 스캐너 커버를 먼저 열었다가 닫음."5자
C. 해결 방법필수시도한 해결 방법 또는 알려진 해결 절차"후면 커버 열어 걸린 용지 제거 후 OK 버튼 누름."5자
💡
영문 입력도 OK: 이라크 등 현장에서 영어로 작성해도 됩니다. Groq AI가 자동 분석 후 영어 스크립트로 생성합니다. 번역 결과는 AI 결과 화면에서 확인 가능합니다.

Step 5 — 사진 첨부
에러 화면, 기계 상태, 정상 복귀 상태 사진을 순서대로 첨부합니다. 각 사진은 AI 이미지 생성의 레퍼런스로 활용됩니다.
사진 1 — 에러 화면 PROB-01
PCOS 에러 메시지 화면 화면에 에러 코드·메시지가 표시된 상태를 촬영합니다.
AI가 에러 상황 이미지(PROB-01) 생성 시 참조합니다.
사진 2 — 기계 상태 SOL-01
해결 과정 중 기계 상태 용지를 꺼내는 등 해결 조작 중의 기계 상태를 촬영합니다.
해결 1단계 이미지(SOL-01) 생성에 사용됩니다.
사진 3 — 정상 복귀 SOL-02
정상 대기 상태 화면 에러 해결 후 PCOS가 정상 대기 상태로 돌아온 화면을 촬영합니다.
손이 화면에 보이지 않도록 주의하세요.
✅
사진 없이도 신청 가능: 사진은 선택 사항입니다. 사진이 없으면 AI가 프롬프트만으로 이미지를 생성합니다. 단, 사진이 있을 때 생성 품질이 향상됩니다.

관리자 주요 기능
  • 1
    새로고침 버튼 상단 nav의 "새로고침" 버튼을 누르면 submit.html에서 새로 들어온 신청을 불러옵니다. 신청은 브라우저 localStorage에 저장되므로 같은 기기에서만 공유됩니다.
  • 2
    영상 프롬프트 확인 신청을 클릭하면 오른쪽 패널에 PROB-01 / SOL-01 / SOL-02 각 클립의 Kling 영상 생성 프롬프트가 표시됩니다. 이 프롬프트를 Higgsfield Kling에 직접 입력하여 영상을 제작합니다.
  • 3
    승인 & 이메일 전송 영상 제작 완료 후 Canva URL을 등록하고 "승인 & 전송" 버튼을 누르면 신청인 이메일로 결과가 발송됩니다.
  • 4
    API 키 설정 (초기 1회) 우측 상단 "PCOS 마스터 설정"에서 Groq API Key를 입력·저장합니다. 저장된 키는 localStorage에 보관되며 서버로 전송되지 않습니다.
?
자주 묻는 질문 (FAQ)
  • Q
    AI 이미지가 생성되지 않으면 어떻게 하나요? Groq API 키가 설정되지 않았을 가능성이 높습니다. admin.html → PCOS 마스터 설정에서 Groq API Key(gsk_...) 를 입력하세요. 그래도 안 되면 관리자(김별하)에게 문의하세요.
  • Q
    11개 PCOS 에러 이외의 에러를 입력해도 되나요? 네. 에러 칩 목록 하단의 "직접 입력"으로 어떤 에러명이든 입력 가능합니다. 단, 11개 프리셋 에러(Paper Jam, UIK Mismatch 등)는 미리 생성된 고품질 이미지를 즉시 사용하고, 그 외는 실시간 AI 생성됩니다.
  • Q
    관리자 페이지에 신청이 안 뜨면? submit.html에서 신청 완료 후 admin.html의 "새로고침" 버튼을 누르세요. submit.html과 admin.html이 같은 브라우저(같은 기기)에서 열려 있어야 합니다. 브라우저가 다르면 localStorage를 공유하지 않습니다.
  • Q
    사진 첨부가 필수인가요? 아닙니다. 사진 없이도 신청 가능합니다. 단, 사진을 첨부하면 AI가 실제 기기 상태를 참조해 더 정확한 이미지를 생성합니다. 특히 SOL-02(정상 복귀) 사진은 "손이 보이지 않는 정면 대기 상태"로 촬영해야 합니다.
  • Q
    에러 유형 CSV 파일은 어디서 확인하나요? 동일 폴더의 error_types.csv 파일을 Excel로 열면 국가·제품별 선택 가능한 전체 에러 유형을 확인할 수 있습니다.
  • Q
    영상은 어떻게 만드나요? admin.html에서 신청을 클릭 → 오른쪽 패널의 "영상 생성 프롬프트"에서 각 클립의 프롬프트를 복사 → Higgsfield Kling/Seedance에 붙여넣어 영상 생성 → Canva에서 편집 → URL을 admin.html의 Canva URL 등록란에 입력 후 "승인 & 전송".
