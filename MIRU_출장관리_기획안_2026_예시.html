01  /  PURPOSE

왜 만들었나

MIRU의 해외 파견 데이터는 기존에 Excel 파일로 관리되었다. 담당자가 직접 데이터를 입력하고 현황을 파악하는 데 매번 수작업이 필요했다.

수작업 집계

통계 확인할 때마다 피벗 테이블 작성

  • 국가별·항공사별·인원별 집계를 매번 별도로 만들어야 함
  • 월별 비용 추이, 좌석 등급 분포 등 시각화 불가
  • 여러 명이 동시 편집 시 버전 충돌 발생
가시성 부재

현황을 한눈에 파악하는 화면이 없음

  • 출장 건수, 총 비용, 방문 국가 수를 즉시 확인 불가
  • 세계 지도에서 파견 빈도를 시각적으로 볼 수 없음
  • 임직원별 출장 비용 비교 분석 불가
접근성 한계

파일 공유 환경의 구조적 제약

  • Excel 파일이 없으면 데이터 조회 자체가 불가
  • 사내 네트워크 어디서나 브라우저로 접속 불가
  • 신규 데이터 추가 후 전체 통계 자동 갱신 안 됨
관리자가 웹에서 직접 데이터를 입력하면 Google Sheets에 자동 저장되고, 대시보드 전체가 실시간으로 갱신된다
  • 데이터 입력 페이지에서 해외 출장 · 한국 방문 · 호텔 예약을 탭으로 구분해 입력
  • 입력 즉시 Google Sheets에 저장 — 대시보드, 출장 목록, 임직원 통계, 예약 가이드에 자동 반영
  • 사내 와이파이 환경에서 브라우저만 있으면 PC·노트북 어디서나 접속 가능

Google Sheets API
  • gspread + google-auth 서비스 계정 인증
  • 해외출장 · 한국방문 · 호텔예약 3개 시트
  • 행 번호 자동 부여 후 append_row 저장
D3.js v7 대시보드
  • 세계 지도 히트맵 (스크롤 줌·드래그 이동)
  • 국가명 → ISO2 코드 자동 매핑 (80개국+)
  • Chart.js 차트 5종 실시간 렌더링
gspread · google-authSheets 연동
  • 서비스 계정 인증: service_account.json 파일로 OAuth 없이 서버 사이드 인증
  • append_row: 헤더 행 탐지 후 번호 자동 부여하여 데이터 행 추가
  • read_sheet: 헤더 기준으로 딕셔너리 형태로 전체 데이터 조회
  • graceful degradation: 키 파일 없을 시 503 반환, 서버 중단 없음
FastAPI + Pydantic백엔드 API
  • 입력 검증: Pydantic BaseModel로 각 탭별 스키마 강제 검증
  • 집계 엔드포인트: Sheets 데이터 읽어 SQLite stats와 동일한 포맷으로 반환
  • 국가명 매핑: KO_TO_ISO2 딕셔너리로 한국어 국가명 → ISO2 자동 변환
  • 로컬 네트워크: --host 0.0.0.0으로 사내 와이파이 전체 접근 허용
D3.js v7 + Chart.js v4프론트엔드
  • 단일 HTML SPA: index.html 한 파일에 관리자 전체 UI 포함
  • 세계 지도 줌/팬: touch-action:none + d3.zoom()으로 스크롤 확대·드래그 이동
  • 데이터 흐름: reloadStats() → /api/sheets/stats → 모든 차트 갱신
  • 폴백 로직: Sheets API 실패 시 로컬 SQLite 데이터로 자동 전환

04  /  HOW TO USE

사용 방법

관리자가 데이터를 입력하고 대시보드에서 확인하는 전체 흐름. 브라우저 하나만 있으면 된다.

🌐
접속 주소 (사내 와이파이)
http://192.168.1.138:8000/admin

데이터 입력 → Google Sheets 저장 → 대시보드 반영

1

사이드바에서 "✎ 데이터 입력" 클릭

왼쪽 사이드바 하단의 데이터 입력 메뉴를 클릭한다. 해외 출장 · 한국 방문 · 호텔 예약 3개 탭 카드가 표시된다.

2

탭 선택 — 해외 출장 / 한국 방문 / 호텔 예약

입력할 유형에 맞는 탭을 클릭한다. 각 탭은 해당 유형에 맞는 입력 필드로 전환된다. 해외 출장은 항공권료·발권수수료·여행자보험, 한국 방문은 도착·출국 일정, 호텔 예약은 체크인·체크아웃 정보를 입력한다.

3

폼 작성 — 우측 패널에서 실시간 확인

좌측 폼을 채울수록 우측 패널이 자동 업데이트된다. 필수 항목에 녹색 체크 표시, 비용 합계 자동 계산, 출장 기간 자동 표시. 모든 필수 항목 입력 완료 시 제출 버튼이 활성화된다.

4

"Google Sheets에 저장" 버튼 클릭

제출 버튼을 클릭하면 FastAPI 서버를 통해 Google Sheets API로 데이터가 전송되고 해당 시트에 새 행이 추가된다. 번호는 기존 데이터 수 + 1로 자동 부여된다.

5

대시보드 · 출장 목록 · 임직원 통계 · 예약 가이드 자동 반영

저장 완료 후 각 페이지로 이동하면 방금 입력한 데이터가 모든 차트·통계·목록에 반영되어 있다. 새로고침 없이도 다음 접속 시 최신 Sheets 데이터를 읽어온다.

해외 출장 입력 항목

출장자 정보: 이름 · 직급 · 소속

출장지 정보: 출장 국가 · 출장 도시 · 항공사 · 좌석 등급

발권 정보: 변경/신규 발권 구분

비용 정보: 항공권료 · 발권수수료 · 여행자보험료 (자동 합산)

일정: 출국일 · 출국시간 · 귀국일 · 귀국시간 (기간 자동 계산)

비고: 선택 입력 메모

한국 방문 / 호텔 예약 입력 항목

한국 방문 — 출장자: 이름 · 국가 · 소속 · 직급

한국 방문 — 항공: 항공사 · 좌석 등급 · 발권 유형

한국 방문 — 비용: 항공권발권료 · 발권수수료 (자동 합산)

한국 방문 — 일정: 도착일 · 도착시간 · 출국일 · 출국시간

호텔 예약: 이름 · 국가 · 소속 · 직급 · 도시 · 호텔명 · 룸 등급

호텔 일정: 체크인 · 체크아웃 날짜 (박수 자동 계산)

05  /  DEPLOYMENT

배포 및 설정

현재 사내 와이파이 기반으로 운영 중. Google Sheets 서비스 계정 키 파일만 받으면 전체 연동이 완성된다.

현재 운영 · Phase 1

사내 와이파이 공유

서버 PC에서 uvicorn 실행, 동일 네트워크 내 누구나 http://192.168.1.138:8000/admin 으로 접속 가능. 별도 비용 없음, 즉시 배포.

운영 중
진행 중 · Google Sheets

Sheets API 연동 완성

IT 담당자에게 서비스 계정 JSON 키 파일 발급 요청 중. 파일을 받아 hotel_dashboard/service_account.json 으로 저장하고 Google Sheet에 서비스 계정 이메일을 편집자로 공유하면 완성.

키 파일 수령 대기 중
다음 단계 · Phase 2

클라우드 배포 검토

Railway.app 또는 사내 서버에 배포해 사외 접속 가능 환경 구성. HTTPS URL 확보 및 접근 권한 관리 추가 예정.

안정화 이후 검토 예정

2JSON 파일 수령 → hotel_dashboard/service_account.json 으로 저장
